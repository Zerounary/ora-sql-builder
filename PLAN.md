# 【一、系统整体定位与核心要求】
### 1. 核心能力
- **企业标准统一元数据层**：统一管理表定义、字段定义、表关联关系、约束、索引、数据源配置
- **元数据搜索模块**：支持按表名、字段名、业务域、数据源、关联关系全文检索
- **跨库通用SQL自动生成引擎**：根据元数据**动态生成标准CRUD SQL**（单表/关联查询、新增、修改、删除、批量操作）
- **通用CRUD执行服务**：封装SQL生成+执行，上层应用零SQL开发
- **上层应用扩展**：基于核心模块快速搭建低代码平台、数据管理后台、开放API

### 2. 硬性技术约束
- 完全**元数据驱动**，无硬编码SQL
- 跨数据库**方言无关**，通过适配器模式兼容各关系库
- 支持**单表/关联表**自动JOIN查询，关联关系由元数据配置驱动
- SQL**参数化**，严防SQL注入
- 支持逻辑删除、非空校验、唯一校验、主键策略（自增/序列/雪花ID）
- 支持多数据源动态切换

### 3. 输出技术栈（Rust专属）
- 后端框架：**Axum**（Web框架） + **Tokio**（异步运行时）
- 数据库连接：**sqlx**（异步SQL执行，支持编译时SQL检查）
- ORM/SQL生成：自定义SQL构建器（基于sqlx）
- 序列化：**Serde** + **JSON**
- 配置管理：**config**
- 日志：**tracing** + **tracing-subscriber**
- 缓存：**Redis**（可选，提升元数据查询性能）
- 接口：RESTful API + OpenAPI文档（**utoipa**）
- 输出内容：中文，结构清晰，代码可直接编译运行
